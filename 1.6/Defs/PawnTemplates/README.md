# Pawn Portability — Character Library

The Pawn Portability system lets you define reusable characters as `PawnTemplateDef` XML and spawn them in-game with full narrative metadata. Characters survive across series: export a memorable colonist from one playthrough, add them to the library, and bring them back in a future series as a colonist, raider, mercenary, or quest pawn.

## Quick start

### Exporting a pawn from in-game

1. **Gizmo (recommended):** Enable "Show Export Pawn gizmo on colonists" in Mod Settings > Pawn Portability. Select a colonist or prisoner and click the Export button in their gizmo bar.
2. **Debug action:** Open the debug actions menu (Development mode) > Pawn Portability > "Export Pawn (Template)", then click a pawn on the map.

Both methods write XML to `{RimWorld save folder}/ExportedPawns/{defName}.xml` (spaces in the defName are replaced with underscores).

Copy the exported `<MSSFP.PawnPortability.Defs.PawnTemplateDef>` block into a file in this directory (e.g. `MSSFP_Pawns.xml`), then fill in the metadata fields (`originSeries`, `tags`, `narrativeNotes`).

### Spawning a character

- **Debug actions:** Pawn Portability > "Spawn From Def" or "Spawn From Def (As Raider)". Pick a def, then click the map cell to place them.
- **C# API:** `PawnPortability.Spawn(defName, position, map, faction, kindOverride)`

### Validating

- **Debug action:** Pawn Portability > "Validate Def". Logs missing mods, missing defs, and fallback decisions.
- **C# API:** `PawnPortability.Validate(def)` returns a `ValidationReport`.

## XML schema

Every file in this directory is loaded as standard RimWorld Defs. The root element is `<Defs>` containing one or more `<MSSFP.PawnPortability.Defs.PawnTemplateDef>` entries.

### Minimal example

```xml
<MSSFP.PawnPortability.Defs.PawnTemplateDef>
    <defName>MSSFP_Pawn_Example</defName>
    <label>Example Character</label>
    <description>A short description.</description>
    <schemaVersion>1</schemaVersion>
    <mode>Template</mode>

    <originSeries>My Series</originSeries>
    <tags><li>mercenary</li></tags>
    <narrativeNotes>Why this character matters to the audience.</narrativeNotes>

    <kindDef>Colonist</kindDef>
    <gender>Male</gender>

    <name>
        <first>John</first>
        <nick>Johnny</nick>
        <last>Smith</last>
    </name>

    <story>
        <bodyType>Male</bodyType>
        <traits>
            <li><def>Tough</def><degree>0</degree></li>
        </traits>
    </story>
</MSSFP.PawnPortability.Defs.PawnTemplateDef>
```

### Full field reference

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `defName` | string | yes | Unique identifier. Convention: `MSSFP_Pawn_{FirstName}` |
| `label` | string | yes | Human-readable display name |
| `description` | string | no | Tooltip / long description |
| `schemaVersion` | int | yes | Always `1` for now |
| `mode` | enum | yes | `Template` (identity overlay) or `Snapshot` (full state, Phase 2) |
| `requiredMods` | list | no | Informational mod manifest (auto-generated by export) |
| `originSeries` | string | no | Which YouTube series this character is from |
| `tags` | list of string | no | Queryable tags (e.g. `mercenary`, `villain`, `ally`) |
| `narrativeNotes` | string | no | Free-text notes for content planning |
| `kindDef` | PawnKindDef | no | Defaults to `Colonist`. Overridden by `kindOverride` at spawn time |
| `gender` | Gender | no | `Male`, `Female`, or `None` |
| `name` | PawnNameData | no | `first`, `nick`, `last` |
| `age` | PawnAgeData | no | `biologicalAgeTicks`, `chronologicalAgeTicks` |
| `story` | PawnStoryData | no | See story fields below |
| `skills` | list | no | `def` (SkillDef), `level` (int), `passion` (`None`/`Minor`/`Major`) |
| `genes` | PawnGeneData | no | `xenotype`, `xenotypeName`, `endogenes`, `xenogenes` |
| `hediffs` | list | no | `def` (HediffDef), `severity`, `bodyPart`, `bodyPartLabel`, `permanent` |
| `equipment` | list | no | Weapons — `def`, `stuff`, `quality`, `hitPoints`, `color`, `stackCount` |
| `apparel` | list | no | Clothing — `def`, `stuff`, `quality`, `hitPoints`, `color`, `stackCount` |
| `inventory` | list | no | Carried items — `def`, `stuff`, `quality`, `hitPoints`, `color`, `stackCount`. **Note:** inventory is not yet applied during spawning (Phase 2) |
| `abilities` | list of AbilityDef | no | Ability defNames |
| `style` | PawnStyleData | no | `beardDef`, `faceTattoo`, `bodyTattoo` |
| `permanentMemories` | list | no | Phase 2 (Snapshot only) — `def` (ThoughtDef), `stage` |
| `ideology` | PawnIdeoData | no | Phase 2 (Snapshot only) — `certainty`, `ideoName` |
| `royalty` | PawnRoyaltyData | no | Phase 2 (Snapshot only) — `titles` (list of `def`/`faction`), `favor` |
| `workPriorities` | list | no | Phase 2 (Snapshot only) — `def` (WorkTypeDef), `priority` |

#### Story fields

| Field | Type | Description |
|-------|------|-------------|
| `bodyType` | BodyTypeDef | `Male`, `Female`, `Hulk`, `Thin`, `Fat`, `Child` |
| `headType` | HeadTypeDef | e.g. `Male_AverageWide` |
| `hairDef` | HairDef | e.g. `Elder`, `Lackland` |
| `hairColor` | Color | RGBA tuple: `(0.279, 0.223, 0.167)` |
| `skinColorOverride` | Color? | Optional skin colour override |
| `furDef` | FurDef | For furry xenotypes |
| `favoriteColor` | ColorDef | e.g. `Structure_Blue`, `Purple` |
| `childhood` | BackstoryDef | Childhood backstory defName |
| `adulthood` | BackstoryDef | Adulthood backstory defName |
| `birthLastName` | string | Maiden name / birth surname |
| `traits` | list | `def` (TraitDef), `degree` (int) |

## MayRequire for cross-mod safety

Any Def-typed XML field or list entry can use the `MayRequire` attribute to gracefully degrade when a mod is absent:

```xml
<xenotype MayRequire="Ludeon.RimWorld.Biotech">Baseliner</xenotype>
<li MayRequire="ludeon.rimworld.anomaly">Leather_Dread</li>
```

When the referenced mod is not active, RimWorld resolves the Def to `null`. At load time, `PawnTemplateDef.ResolveReferences()` automatically:
- Removes list entries where the primary Def is null
- Applies stuff material fallbacks (via `GenStuff.DefaultStuffFor`)
- Falls back to `Baseliner` xenotype if Biotech is active but no xenotype is specified
- Falls back to gendered body type in Snapshot mode if missing

The exporter auto-generates `MayRequire` attributes for all non-Core defs.

## Template mode spawning

Template mode generates a fresh pawn via `PawnGenerationRequest`, then **overlays** only the fields present in the def. This means:

- Null/missing fields are left at whatever PawnGenerator rolled
- Traits, genes, and hediffs are **replaced entirely** (cleared, then set from template)
- Skills are **set** (only skills listed in the def are touched; others keep generated values)
- Abilities are **added on top** of any gene-sourced abilities
- Equipment and apparel are **replaced entirely**

This design means a template with only `name`, `gender`, and `traits` still produces a complete pawn — PawnGenerator fills in everything else.

## C# API

The `PawnPortability` static class is the single public entry point:

```csharp
using MSSFP.PawnPortability;
using MSSFP.PawnPortability.Defs;

// Export
PawnPortability.Export(pawn, filePath, PawnTemplateMode.Template);
string xml = PawnPortability.ExportToString(pawn, PawnTemplateMode.Template);

// Spawn
Pawn p = PawnPortability.Spawn("MSSFP_Pawn_Caius", position, map);
Pawn raider = PawnPortability.Spawn(def, position, map, enemyFaction, PawnKindDefOf.Pirate);

// Create without placing on map
Pawn p = PawnPortability.Create("MSSFP_Pawn_Caius", Faction.OfPlayer);

// Query
IEnumerable<PawnTemplateDef> mercs = PawnPortability.GetDefsByTag("mercenary");
IEnumerable<PawnTemplateDef> series = PawnPortability.GetDefsBySeries("Morrowind Run");
PawnTemplateDef def = PawnPortability.GetDef("MSSFP_Pawn_Caius");

// Validate
ValidationReport report = PawnPortability.Validate(def);
// report.CanSpawn, report.IsFullFidelity, report.MissingMods, report.MissingItems, report.Fallbacks

// Duplicate prevention
bool alive = PawnPortability.IsAlive(def);
Pawn existing = PawnPortability.FindLivePawn(def);
```

## Hediff export rules (Template mode)

The exporter includes hediffs where:
- `def.isBad == false` (implants, bionics, prosthetics), OR
- `IsPermanent() == true` (scars — the hediff has a `HediffComp_GetsPermanent` comp with `IsPermanent` set)

Fresh wounds, diseases, and temporary conditions are excluded. To export a scar, the injury must have fully healed into a permanent scar (shows as "stab (scar)" etc. in the health tab) before exporting.

Enable "Enable Pawn Export/Import Debug Logging" in settings to see detailed diagnostics of which hediffs pass or fail the filter.

## Settings

Found under Mod Settings > Mr Samuel Streamer Flavour Pack > Pawn Portability:

| Setting | Default | Description |
|---------|---------|-------------|
| Show Export Pawn gizmo | Off | Adds an Export button to the gizmo bar of player-controlled colonists and prisoners |
| Enable Pawn Export/Import Debug Logging | Off | Verbose logging for export, import, spawning, and validation |

## File organisation

Place XML files in this directory. RimWorld loads all `*.xml` files as Defs automatically. You can use a single file (`MSSFP_Pawns.xml`) or split by series/category — whatever keeps the library manageable.

Convention for `defName`: `MSSFP_Pawn_{FirstName}` for templates, `MSSFP_Snapshot_{FirstName}_{TicksAbs}` for snapshots.
